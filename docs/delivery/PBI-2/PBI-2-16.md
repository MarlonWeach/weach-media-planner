# Task 2-16: Pré-selecionar solicitante da sessão em nova cotação

## Status
✅ Done

## Descrição
Na tela de nova cotação, pré-selecionar automaticamente o "Nome do Solicitante" com base no usuário autenticado na sessão, mantendo o campo livre para troca manual quando necessário.

## Objetivo
Reduzir atrito no preenchimento inicial da cotação para usuários comerciais, sem perder flexibilidade operacional em casos de apoio entre pares.

## Critérios de Aceite
- [x] Em `/cotacao/nova`, o solicitante é pré-selecionado quando o e-mail da sessão encontra correspondência no cadastro de solicitantes.
- [x] O campo `Nome do Solicitante` permanece editável e permite trocar para outro solicitante.
- [x] A pré-seleção não sobrescreve uma seleção já existente no formulário.

## Implementação
- Atualizado `components/cotacao/WizardStep1.tsx` para buscar `/api/auth/me` no carregamento das opções do formulário e cruzar o e-mail do usuário logado com a lista de solicitantes.
- Quando houver correspondência e não existir seleção prévia, o formulário define `solicitanteId`, `solicitanteNome` e `solicitanteEmail`.
