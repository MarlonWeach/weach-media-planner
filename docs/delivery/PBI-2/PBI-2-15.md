# Task 2-15: Reenviar convite/reset de senha do solicitante em /admin/cadastros e garantir provisionamento COMERCIAL automático

## Status
✅ Done

## Descrição
Adicionar ação de reset de senha em `/admin/cadastros` que reenviará convite de definição de senha para o solicitante. Garantir que, ao concluir a definição de senha, o solicitante passe a existir na gestão de usuários com role `COMERCIAL`.

## Objetivo
Permitir recuperação operacional de acesso para solicitantes e consolidar automaticamente o usuário na base de acesso.

## Critérios de Aceite
- [x] Existe botão `Reset senha` por solicitante em `/admin/cadastros`.
- [x] O botão dispara novo convite de definição/redefinição de senha por e-mail.
- [x] Fluxo de criação/redefinição mantém token seguro e expirável.
- [x] Ao concluir definição de senha, o usuário é criado/atualizado e aparece em `/admin/usuarios`.
- [x] Role final do usuário após onboarding é `COMERCIAL`.

## Implementação
- Criada rota `POST /api/admin/solicitantes/[id]/onboarding` para emissão de novo token e reenvio de e-mail.
- Tela `app/admin/cadastros/page.tsx` recebeu botão `Reset senha` na lista de solicitantes.
- Mensagem de sucesso da criação de solicitante foi atualizada para deixar explícito o envio do convite por e-mail.
- API de conclusão de onboarding (`/api/auth/complete-onboarding`) foi reforçada para definir role `COMERCIAL` também quando usuário já existir.
