# Task 2-12: Implementar gestão de usuários de acesso integrada ao cadastro base

## Status
✅ Done

## Descrição
Criar fluxo administrativo para provisionar e gerenciar usuários de acesso à plataforma com base no cadastro base (principalmente solicitantes), incluindo definição de perfil e ciclo de vida de acesso.

## Objetivo
Permitir controle centralizado de usuários com vínculo operacional ao cadastro base, reduzindo criação manual desconectada e melhorando governança de acesso.

## Roles da Task
- `ADMIN`
- `MANAGER`
- `COMERCIAL`

## Escopo
- Criar usuário de acesso a partir de cadastro base (solicitante).
- Definir/alterar role de usuário (`ADMIN`, `MANAGER`, `COMERCIAL`).
- Ativar/Inativar usuário.
- Evitar duplicidade de usuário por e-mail.
- Exibir listagem e filtros básicos de usuários no painel admin.

## Critérios de Aceite
- [x] Admin consegue criar usuário a partir de um solicitante existente.
- [x] Sistema bloqueia criação duplicada para mesmo e-mail.
- [x] Admin consegue alterar role entre `ADMIN`, `MANAGER` e `COMERCIAL`.
- [x] Admin consegue ativar/inativar usuário sem excluir histórico.
- [x] Tela de gestão de usuários possui listagem e filtros mínimos.

## Observações Técnicas
- Atualmente o enum de role no banco/código usa `ADMIN`, `COMERCIAL`, `EXTERNO`.
- Esta task inclui avaliação de migração para introduzir `MANAGER` sem quebrar compatibilidade.
