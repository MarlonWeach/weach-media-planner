# Task 2-13: Implementar menu de usuário (ajustes e troca de senha) e reforçar bloqueio de acesso comercial ao /admin

## Status
✅ Done

## Descrição
Criar um menu de perfil do usuário no topo da aplicação (avatar/nome) com acesso a ajustes da conta, incluindo alteração de senha. Em paralelo, revisar e reforçar o controle de acesso ao módulo administrativo para impedir acesso de usuários com perfil `COMERCIAL`.

## Objetivo
Melhorar a experiência de autoatendimento de conta e garantir governança de permissões no painel administrativo.

## Escopo
- Exibir botão/menu de usuário semelhante ao padrão de avatar com dropdown.
- Disponibilizar opção de ajustes do próprio usuário.
- Incluir fluxo de alteração de senha para o usuário autenticado.
- Revisar restrições de acesso à rota `/admin` (frontend e backend) para bloquear `COMERCIAL`.

## Critérios de Aceite
- [x] Usuário autenticado visualiza botão/menu de perfil no layout principal.
- [x] Ao clicar no menu, existem opções mínimas de ajustes da conta e alteração de senha.
- [x] Fluxo de troca de senha valida senha atual e confirma nova senha.
- [x] Usuário `COMERCIAL` não consegue acessar `/admin` via navegação direta.
- [x] Endpoints administrativos também bloqueiam `COMERCIAL`, mesmo com tentativa manual de chamada.
- [x] Comportamento de `ADMIN` e `MANAGER` permanece estável e sem regressão.

## Observações
- Priorizar padrão visual já existente na aplicação para evitar divergência de UI.
- Garantir mensagens claras de erro/permissão para evitar confusão operacional.

## Implementação
- Criado componente `components/layout/UserMenu.tsx` com avatar, acesso a `Ajustes da Conta` e ação de logout.
- Integrado `UserMenu` em `app/dashboard/page.tsx` e `app/admin/page.tsx`.
- Criada página `app/conta/ajustes/page.tsx` com formulário de troca de senha para usuário autenticado.
- Criada API `POST /api/auth/change-password` em `app/api/auth/change-password/route.ts`, com validação de senha atual, confirmação de nova senha e persistência segura de hash.
- Reforçado bloqueio de acesso ao `/admin` para perfil `COMERCIAL` no frontend da página administrativa.
