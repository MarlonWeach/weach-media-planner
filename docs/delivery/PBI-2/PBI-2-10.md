# Task 2-10: Corrigir aplicação de regionalização (Nacional x Estado x Cidade) na precificação

## Status
✅ Done

## Descrição
Corrigir o cálculo de preço para que a regionalização selecionada na cotação seja efetivamente aplicada no motor de pricing, evitando fallback indevido para valores de `NACIONAL` quando o contexto for `ESTADOS`, `CIDADES_GRANDES` ou `CIDADES_MENORES`.

## Contexto do Problema
Teste funcional indicou divergência: cotações por estado/cidade estão saindo com valores equivalentes ao cenário nacional.

## Objetivo
Garantir aderência da regra de negócio de contexto regional (referência da planilha) em toda a cadeia de precificação exibida no plano de mídia, PDF e e-mail.

## Critérios de Aceite
- [x] Cotação com regionalização `NACIONAL` usa tabela/multiplicador nacional.
- [x] Cotação com regionalização de estado/cidade usa tabela/multiplicador correspondente.
- [x] Fallback para nacional só ocorre quando regra regional realmente não existir.
- [x] Endpoint/fluxo de criação de cotação retorna preços coerentes com a regionalização selecionada.
- [x] Evidência de teste manual com pelo menos 3 cenários (nacional, estado, cidade).
