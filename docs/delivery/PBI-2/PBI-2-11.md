# Task 2-11: Validar reconhecimento de cidades e alertar correção quando não identificada

## Status
✅ Done

## Descrição
Implementar uma checagem de identificação da cidade informada no campo de cobertura. Quando a cidade não for reconhecida pelo mecanismo de classificação (alias/fuzzy), o sistema deve apontar claramente essa situação para o usuário revisar e corrigir.

## Objetivo
Evitar classificação incorreta por cidade digitada com erro, ambiguidade ou ausência no mapeamento, reduzindo risco de precificação regional equivocada.

## Critérios de Aceite
- [x] Ao informar cidade não reconhecida, o sistema exibe alerta no campo com mensagem orientativa.
- [x] A mensagem indica quais cidades não foram identificadas.
- [x] O usuário consegue corrigir e reenviar sem perder os demais dados do formulário.
- [x] Cidades reconhecidas continuam seguindo a lógica de classificação atual.
- [x] O comportamento é aplicado para uma ou múltiplas cidades no campo.

## Escopo Técnico Inicial
- Revisar parser de cidades no fluxo de cotação.
- Expor retorno estruturado de cidades não reconhecidas.
- Exibir feedback no frontend (campo de cidade/cobertura).
- Garantir que o alerta não bloqueie indevidamente outros fluxos válidos.
