# Task 9-10: Dashboard — tag Programática / Performance / Mensageria

## Status
✅ Done

## Descrição
Na listagem de cotações em `/dashboard`, exibir uma **tag** (ou chip) por linha indicando o tipo principal da cotação:

- **Programática** — quando a definição incluir escopo programático (ex.: `PROGRAMATICA` / mix alinhado).
- **Performance** — quando a definição for performance ou híbrido com performance (reutilizar regras já usadas no wizard, ex. `resolverApenasPerformance` / texto em observações).
- **Mensageria** — quando escopo for CRM/WhatsApp/SMS/Push ou equivalente documentado no projeto (`WHATSAPP_SMS_PUSH` ou similar).

## Objetivo
Permitir triagem visual rápida na fila operacional sem abrir cada cotação.

## Critérios de Aceite
- [ ] Cada linha da lista exibe exatamente uma tag primária (ou regra documentada para híbrido: ex. duas tags ou tag composta, a definir no implementação).
- [ ] A regra de mapeamento é a **mesma fonte de verdade** usada em fluxos já existentes (evitar divergência com e-mail/PDF).
- [ ] Tag acessível (cor + texto, não só cor).

## Notas
- Se houver cotação “mista”, documentar no PR/commit se a tag prioriza performance, programática ou mostra “Múltiplo”.
